# 子資料夾不會被上傳的解法

GitHub 網頁的 **Upload files** 在拖曳「資料夾」時，**不會**把資料夾裡的檔案一層層傳上去，所以會變成只有根目錄的檔案，沒有 `pages/`、`db/` 等子資料夾的內容。

有兩種做法：

---

## 做法一：用網頁「Create new file」手動建路徑（純網頁）

GitHub 的規則是：**檔名裡寫路徑，就會自動建立資料夾**。

### 步驟（以建立 `pages/1_交易輸入.py` 為例）

1. 在 repo 頁面點 **Add file** → **Create new file**
2. 在 **檔名** 欄位**不要**只打 `1_交易輸入.py`，要打完整路徑：  
   **`pages/1_交易輸入.py`**  
   一輸入 `pages/`，上方就會出現「Create new file in pages」之類的提示，表示會建立 `pages` 資料夾。
3. 下方編輯區貼上該檔案的完整內容（從本機 `stockanalysis\pages\1_交易輸入.py` 複製）。
4. 捲到最下面按 **Commit new file**。
5. 其他檔案重複同樣方式，例如：
   - `pages/2_Portfolio持倉與損益.py`
   - `pages/3_日成交彙總.py`
   - `pages/4_投資績效.py`
   - `pages/5_主檔設定.py`
   - `db/__init__.py`
   - `db/models.py`
   - `db/database.py`
   - `db/seed_data.py`
   - `services/__init__.py`
   - `services/price_service.py`
   - `services/pnl_engine.py`
   - `reports/__init__.py`
   - `reports/portfolio_report.py`
   - `reports/daily_summary.py`
   - `tests/__init__.py`
   - `tests/test_pnl_engine.py`

這樣子資料夾就會一個一個被建出來，且檔案都會在正確路徑下。缺點是要重複很多次（約 18 個檔案）。

---

## 做法二：用 GitHub Desktop 只做「這一次」上傳（推薦）

子資料夾上傳是 **GitHub 網頁的限制**，用 **GitHub Desktop** 或 **git 指令**可以一次把整個專案（含所有子資料夾）推上去，**之後**要改單一檔案還是可以回 GitHub 網頁編輯。

### 步驟簡述

1. 安裝 **Git**：https://git-scm.com/download/win  
2. 安裝 **GitHub Desktop**：https://desktop.github.com/  
3. 在 GitHub Desktop：**File** → **Clone repository** → URL 填 `https://github.com/你的帳號/stocktest`，Local path 選例如「桌面」，按 **Clone**（會出現本機的 `stocktest` 資料夾）。
4. 用檔案總管把 **stockanalysis** 裡面的**所有檔案與資料夾**（Ctrl+A 全選）**複製**，貼到剛 Clone 下來的 **stocktest** 資料夾裡，有重複的選「取代」。
5. 回到 GitHub Desktop，左邊會出現很多 Changes，下面寫 commit 訊息後按 **Commit to main**，再按 **Push origin**。

這樣 `pages`、`db`、`services`、`reports`、`tests` 都會一次上傳，之後若要改一兩個檔，可以繼續用網頁 **Edit** 即可。

---

## 總結

| 方式 | 子資料夾 | 適合情況 |
|------|----------|----------|
| 網頁 Upload files（拖曳資料夾） | ❌ 不會一起上傳 | 只適合根目錄的單一檔案 |
| 網頁 Create new file，檔名打 `資料夾/檔名` | ✅ 會建立資料夾 | 願意手動建約 18 個檔案時 |
| GitHub Desktop / git 推整個專案 | ✅ 一次全部上傳 | 建議至少用這一次，之後可繼續用網頁改 |

若你希望「從此只用網頁」，就用做法一逐檔建立；若可以接受「裝一次 Desktop 做完上傳」，建議用做法二，比較省時。
