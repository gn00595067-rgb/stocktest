# 網頁板沒看到更新 — 檢查清單

「網頁板」可能指：(1) **GitHub 網站**上的 repo 沒看到新檔案；(2) **Streamlit Cloud 上的 App** 沒顯示最新程式。依情況對照下面步驟。

---

## 情況一：GitHub 網站上沒看到更新

你已經用 GitHub Desktop 或 git Push 了，但打開 https://github.com/你的帳號/stocktest 還是沒看到新資料夾或新檔案。

### 請依序檢查：

**1. 有沒有按「重新整理」**  
- 按 **F5** 或 **Ctrl + R** 重新整理頁面  
- 或關掉分頁再重新打開 repo 網址  

**2. 看的是不是 main 分支**  
- 頁面左上角有一個分支選單，寫 **「main」** 或 **「master」**  
- 點一下確認選的是 **main**（你 Push 的通常是 main）  
- 若選到別的分支，就會看不到剛 Push 的內容  

**3. 網址是不是這個 repo**  
- 確認網址是：`https://github.com/你的帳號/stocktest`  
- 若有多個 repo，不要開錯另一個  

**4. Push 有沒有成功**  
- **GitHub Desktop**：Push 後上方若沒寫錯誤，通常就是成功；再看一次 **Repository** → **View on GitHub** 會打開網頁  
- **Git 指令**：若 `git push` 有出現 `Everything up-to-date` 或沒有報錯，就表示有送上去  

**5. 強制重新整理（排除快取）**  
- **Ctrl + Shift + R**（Windows）或 **Ctrl + F5**  
- 或改用無痕視窗開同一個 repo 網址看一次  

若以上都確認過還是沒有，請看 GitHub Desktop 左上方有沒有「Push origin」可以按；若還有 1 commit 沒推，按一次 **Push origin** 再重新整理網頁。

---

## 情況二：Streamlit Cloud 上的 App 沒更新

你是改 GitHub 上的程式（或本機 Push 上去），但打開 **Streamlit Cloud 的 App 網址**（例如 https://xxx.streamlit.app）還是舊畫面或舊功能。

### 原因說明

Streamlit Community Cloud 是「從 GitHub 拉程式再執行」。你改 GitHub 後，**Cloud 不一定會馬上自動重新部署**，所以網頁板會暫時還是舊版。

### 做法：手動觸發重新部署

**方法 A：在 Streamlit Cloud 網頁按 Redeploy**

1. 打開 **https://share.streamlit.io** 並登入  
2. 點進你的 **stocktest** 這個 app  
3. 右上角選單（三個點 **⋮**）或 **Manage app** → 找 **「Reboot app」** 或 **「Redeploy」**  
4. 按下去後等 1～3 分鐘，再開你的 App 網址就會是新版  

**方法 B：空 commit 觸發重新部署**

1. 到 GitHub repo：**https://github.com/你的帳號/stocktest**  
2. 隨便開一個檔（例如 **README.md**）  
3. 點 **Edit（鉛筆圖示）**  
4. 在內容最後加一個空行或一個字，再按 **Commit changes**  
5. Streamlit Cloud 偵測到 repo 有新的 commit，通常會自動重新部署；等幾分鐘再開 App 網址  

**方法 C：確認 App 設定**

- 到 **https://share.streamlit.io** → 你的 app → **Settings**  
- 確認 **Branch** 是 **main**  
- 確認 **Main file path** 是 **app.py**  
- 若之前選錯分支或主檔，改完存檔後也會觸發重新部署  

---

## 總結

| 狀況 | 可能原因 | 處理方式 |
|------|----------|----------|
| GitHub 網頁沒看到新檔案/資料夾 | 沒重新整理、看錯分支、Push 沒成功 | F5、確認 main、再 Push 一次、Ctrl+Shift+R |
| Streamlit App 網頁板沒更新 | Cloud 還沒重新部署 | share.streamlit.io 裡 Reboot/Redeploy，或對 repo 做一次小改動再 Commit |
